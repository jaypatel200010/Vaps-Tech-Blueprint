Strategic Architecture Proposal

**Vaps Tech** currently operates an **Education ERP** managing admissions, courses, and student structures. Rather than disrupting that core operational system, **nexaflowtech** proposes deploying **Frappe ERPNext** to act as the master engine for all other business domains: **Multi-channel Sales/E-commerce, Recruitment, Social Media Chat, and Video Subscriptions**.

This blueprint web application provides an interactive walk-through of the design workflows and data mappings. By clicking on specific **Doctype badges** inside any section, you can review the database schemas that will be deployed.

High-Performance Rollout: ERPNext's lightweight database structure, event-driven socket pipelines, and built-in portal modules allow this architecture to scale efficiently without requiring massive custom middleware codebases.

Implementation Roadmap & Phased Timeline

Phase 1

ERPNext Core Setup & HR Recruitment

Deploy base Frappe instance. Enable ERPNext native web portal framework. Set up organizational structures and launch the native Job Board for candidates.

Phase 2

E-Commerce Multi-Channel API Connection

Establish secure integration lines with Shopify, Flipkart, and Amazon. Build stock-level synchronization logic and automate stock reservation upon checkout.

Phase 3

Social Media Chat & Direct Messages

Implement WebSockets (socket.io) on port 9000 for real-time messaging between user profiles, storing communication logs in custom Direct Message Doctype tables.

Phase 4

Live Video Stream Billing & Subscriptions

Set up Subscription Plans and link billing to automated invoicing. Connect the external video player's authorization flow to our ERPNext verification API.

Sales Order & Inventory

Process Flow Description:

  • Order Placement: Customer places order on Shopify, Flipkart, or Amazon. An API webhook triggers immediately.
  • Validation & Allocation: ERPNext intercepts the JSON webhook, verifies authenticity, reserves physical items in the warehouse, and records a Sales Order.
  • Stock Sync: Stock updates (decreases) trigger a push payload back to all sales channels to update their live inventory counts.
  • Reconciliation: Payments and refunds process dynamically, automatically writing corresponding Sales Invoice and Payment Entry forms.

In-Built Native Job Portal Flow

Rather than building a complex standalone job portal that requires manual data synchronization, we deploy **ERPNext's native recruitment and job board modules**. This leverages built-in web views to map candidate applications directly into HR workflows.

Staff Onboarding

Process Flow Description:

  • Creating Vacancy: The HR team publishes a Job Opening directly in ERPNext. Setting 'Publish on Website' automatically displays the position on the built-in portal (/jobs).
  • Application: Candidates fill out the job application form, upload resume files, and submit. The entry writes directly to the ERPNext database as a Job Applicant.
  • Evaluation: HR moves applicants through a pre-defined workflow (Applied -> Shortlisted -> Interview -> Hired).
  • Hiring: Upon hiring approval, a Job Offer generates. Acceptance triggers automated creation of the Employee card, initializing onboarding tasks.

Social Media Chat & Direct Messaging

Enables social network-style communications inside the application. Users can set up a social profile, search for other users, and engage in private one-on-one chats (DMs). This provides direct, real-time connectivity between system subscribers, students, and employees.

Direct Message

Process Flow Description:

  • Profile Discovery: Users establish a public Social Profile inside the network, enabling search, discovery, and connection permissions.
  • DM Room Hook: Initiating a direct message constructs a private 1-to-1 DM Room linking the sender and receiver profiles securely.
  • WebSocket Delivery: Active chat messages route in real-time using socket.io over port 9000 and write instantly to the Direct Message database node.
  • Privacy & Safety: Messages are strictly constrained to 1-to-1 channels, preventing group message leakages, and logs sync with compliance moderation rules.

Streaming Authorization & Subscriptions

**Vaps Tech** provides live video feeds and content archives. Instead of managing access control within the video platform itself, ERPNext manages subscription billing, while the video server calls a Frappe verification API before authorizing playback.

📺

🌐

💳

ERPNext Engine

Validates Subscription

Process Flow Description:

  • Asset Request: A viewer clicks play on a live stream in the browser client. The player requests authorization keys.
  • API Verification Check: The media server hooks into Frappe's verification API endpoint (e.g. `/api/method/streaming.validate_subscription`), passing User ID and Asset SKU.
  • Database Rule Lookup: Frappe queries the active Subscription document associated with the Customer. If the state is 'Active' and payment holds good, it returns 'ALLOWED'.
  • Stream Delivery: If subscription check returns false, the player denies playback and launches a Subscription Billing billing widget.
  • Billing Engine: Recurring invoice schedules generate Sales Invoice transactions in ERPNext at defined intervals.
