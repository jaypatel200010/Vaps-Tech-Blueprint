MODULE 01
01

E-Commerce Integration

Vaps Tech maintains central control of catalog listings, price schedules, and stock accounts inside ERPNext. Products are published outward and sold across Shopify, Flipkart, and Amazon storefronts.

01
Catalog Creation
Products, pricing lists, and item classifications are managed directly inside ERPNext.
02
Outward Push
ERPNext REST APIs transmit catalog item structures and inventory levels to storefronts.
03
Store Checkout
Customer checkout on the storefront triggers instant webhook JSON payloads to ERPNext.
04
Sales Order Draft
ERPNext captures webhook data, generates a Sales Order, and locks the inventory items.
05
Logistics & Sync
ERPNext logs stock deductions upon delivery, pushing revised stock levels back to storefronts.
Control Logic: Back-office invoicing, payment reconciliations, and warehouse ledger entries remain centralized inside **ERPNext** as the source of truth, while Shopify, Amazon, and Flipkart function strictly as storefronts.
MODULE 02
02

Native Job Portal

A native recruitment portal built directly inside ERPNext. Branch managers request hiring budgets, post vacancy listings online, and monitor candidate evaluations from one system.

01
Requisition Draft
Branch staff log vacancy needs, routing them for manager approval inside ERPNext.
02
Portal Vacancy
Approved requisitions populate Vaps Tech's native Job Board for applicant viewing.
03
Candidate Entry
Applicants submit profiles and CVs, writing a Job Applicant record in ERPNext.
04
Interview Track
HR schedules screenings and logs interviewer feedback directly inside the candidate profile.
05
Employee Build
System issues an offer letter, closing the requisition and generating a new Employee file.
Control Logic: Candidates are screened using ERPNext's native recruitment pipelines. This connects recruitment data with salary structures, attendance sheets, and performance appraisals.
MODULE 03
03

Streaming & Subscriptions

Instructors run streaming sessions with student rosters. ERPNext verifies student subscription status, validates access keys, and logs watch-time to calculate teacher coin rewards.

01
Stream Scheduled
Teachers schedule sessions inside ERPNext, mapping them to student classrooms.
02
Token Check
Student requests entry; ERPNext validates subscription status and outputs an access key.
03
Live Broadcast
Video stream runs via WebRTC (Agora/100ms API). Attendance and join times are tracked.
04
Analytics Sync
At stream end, the integration pushes watch duration and view metrics to ERPNext.
05
Reward Ledger
ERPNext audits session data, converting views to coin rewards on the teacher's ledger.
Control Logic: Agora / 100ms SDK acts as the WebRTC video delivery pipe, while **ERPNext** functions as the central logic core managing schedules, student subscription levels, and teacher coin accounts.
MODULE 04
04

Social Community & DMs

A secure social space for student-teacher interaction. Users manage profiles, participate in academic boards, and communicate via secure 1-to-1 direct messaging (DMs).

01
Auto Profile
User profiles are mapped to active Student and Teacher database records in ERPNext.
02
Academic Feeds
Instructors post announcements, slides, questions, or educational topics.
03
Discussion Board
Students and colleagues comment and reply in threads to solve queries.
04
Direct Messages
Secure 1-to-1 chat rooms allow direct messaging between students and instructors.
05
Term Filter
Automatic profanity filters run background flags; admins review reported reports.
Control Logic: Profiles are bound to active ERP credentials. 1-to-1 direct messaging threads are logged for safety compliance.
HOW IT ALL CONNECTS

One identity, shared everywhere

Every module reads from the same student, teacher, and school records already in the ERP. Nothing is duplicated, and nothing needs a second login.

🔑 Single Sign-On (SSO)
A single credentials database auths users across ERPNext, Storefronts, Recruitment Boards, WebRTC stream players, and Social feeds.
📊 Shared Profile Registers
Student, Teacher, and Branch profiles are maintained strictly inside ERPNext as the master source of truth. Every module references this directory.
🧩 Subscription-Aware
System access controls are synchronized. Access to modules (e.g. streaming, social feeds) is tied directly to active ERP subscription statuses.
ROADMAP

Phase-wise delivery milestones

Sequenced so the shared integration foundation is established first, followed by sequential module releases. Live Streaming is allocated the longest duration due to third-party WebRTC SDK verification.

Phase 00 2 Weeks

Discovery & Architecture

Align organizational metadata and document ERPNext deployment configurations. Set up server staging instances and map branch ledgers.

Phase 01 2 Weeks

Core Integration Layer

Establish Single Sign-On (SSO) structures. Build authentication hooks and map core profile directories from the school database.

Phase 02 4 Weeks

E-Commerce Sync

Connect ERPNext inventory modules with external storefronts (Shopify, Flipkart, Amazon). Build stock push APIs and purchase order webhooks.

Phase 03 3 Weeks

Job Portal Board

Deploy native recruitment vacancy boards inside ERPNext. Setup requisition flows, candidate profile submissions, and HR review states.

Phase 04 5 Weeks

Streaming & Subscriptions

Integrate WebRTC video API tokens (Agora/100ms). Configure subscription checking hooks and reward conversion calculations in ERPNext.

Phase 05 4 Weeks

Social Community

Build role-based profile feeds, comment threads, and secure 1-to-1 direct messaging (DM) layers with profanity filtering.

Phase 06 2 Weeks

Integration Testing & Go-Live

Execute security audits, performance tests, User Acceptance Testing (UAT), and migrate server assets to live production.
